# Mobile — Web Parity Plan

Brings the iOS app to feature and visual parity with the web app across three pillars: **Retire / FIRE**, **Home calculators**, and **Money / Budget**. Tracker is out of scope.

Source of truth (web):
- `ui/components/features/calculator/**`
- `ui/components/features/home-calc/**`
- `ui/components/features/money/**`

Target surface (mobile):
- `mobile/src/features/calculator/**` · `mobile/src/features/results/**`
- `mobile/src/features/home-calc/**`
- `mobile/src/features/money/**`
- `mobile/src/components/ui/charts/**`
- `mobile/src/components/layout/TopBar.tsx`
- `mobile/app/(tabs)/*`

---

# Phase 1 — Foundation (complete)

Initial rebuild of the three pillars. All deliverables shipped; mobile typecheck and FIRE engine parity check pass.

- Retire: native tab header hidden; `TopBar` owns Launcher / theme toggle / auth. Advanced FIRE wizard split into per-step files (`StepYou`, `StepIncome`, `StepPortfolio`, `StepAdvanced`, `StepGoals`) with `StepStrip` and shared helpers. Output gating on Simple + Advanced. Results screen rebuilt to match `ResultsDashboard` section order.
- Charts kit: `ChartFrame`, `Tooltip`, `PortfolioAreaChart`, `SensitivityBarChart`, `MonteCarloFanChart`, `YearlyMiniRow`, `MultiAreaChart`, `StackedBarLineChart`, `ScrollableTable`, `DonutChart`.
- Home: `BreakEvenView`, `MortgageView`, `AffordabilityView`, `BuyingGuide` each in their own file; profile picker in BottomSheet with delete; reset-defaults per calc.
- Money: 7-view split (`DailyView`, `CalendarView`, `MonthlyView`, `SummaryView`, `StatsView`, `BudgetView`, `NotesView`); `FAB`; `AddTransactionSheet` split with `PickerSheetRow` + native date picker; `ManageCategoriesSheet`, `ManageAccountsSheet`; reanimated fade between views; CSV export via share sheet.
